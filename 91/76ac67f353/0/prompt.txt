# Plan: vLLM Server Script for Reserved TPU

## Context

We need a CLI script to start and manage a public vLLM inference server on a reserved v4-8 TPU instance (35.186.47.159, us-central2-b). The script SSHes into the TPU, starts a Docker container running vLLM, and provides lifecycle management. Defaults to serving `Qwen/Qwen3-8B`.

## New File

**`scripts/serving/vllm_server.py`** — Click-based CLI with `start`, `stop`, `logs`, `status` subcommands.

## Design

### SSH approach
Direct SSH to the external IP (`ssh user@35.186.47.159 <command>`). Simpler than `gcloud compute tpus tpu-vm ssh` for a known reserved instance. SSH options match `DirectSshRemoteExec` from `lib/iris/src/iris/cluster/platform/remote_exec.py`.

### Docker container management
Builds a `docker run -d --net=host --privileged --shm-size=100gb ...` command (same flags as `_docker_run_args_for_resource()` and `_build_docker_run_command()` in `lib/marin/src/marin/inference/vllm_server.py`), executes it remotely via SSH. Container uses a fixed name (`marin-vllm-serving`) for easy lifecycle management.

### Environment variables
Replicates `_vllm_jax_env()` from `lib/marin/src/marin/inference/vllm_server.py:747-775`: `TOKENIZERS_PARALLELISM`, `MODEL_IMPL_TYPE`, `TPU_MIN_LOG_LEVEL`, `JAX_COMPILATION_CACHE_DIR`, etc. Forwards `HF_TOKEN` for gated model access.

### CLI structure (Click, matching project convention)
- **`start`** — SSH in, remove stale container, start Docker, print endpoint URL, optionally tail logs
- **`stop`** — `docker rm -f` the container remotely
- **`logs`** — View/follow container logs remotely
- **`status`** — Check container state + hit `/v1/models` to confirm vLLM readiness

### Defaults
| Setting | Default |
|---------|---------|
| Model | `Qwen/Qwen3-8B` |
| Port | `8000` |
| Docker image | `vllm/vllm-tpu:nightly-20260104-4a1e25b-0d4044e` |
| Host bind | `0.0.0.0` (public) |
| TPU IP | `35.186.47.159` |
| Container name | `marin-vllm-serving` |

### Key options
`--model`, `--port`, `--docker-image`, `--max-model-len`, `--hf-token` (envvar `HF_TOKEN`), `--extra-args`, `--host` (TPU IP), `--user` (SSH user), `--ssh-key`, `--tail/--no-tail`
