2. Key Technical Concepts:
   - **Student/Teacher SFT**: Fine-tuning student models on teacher-generated agentic trajectories from SWE-smith (Rust/ripgrep coding tasks)
   - **TPU v5p-32**: 16 chips, ~95GB HBM each. Per-device parallelism=1 with batch_size=16.
   - **Gradient checkpointing with carry offload**: `haliax.ScanCheckpointPolicy(save_carries="offload")` — offloads layer hidden states to host memory, essential for fitting 32B model on v5p-32
   - **LR schedule in Levanter**: `warmup` + `stable` + `decay` = total steps. `decay` is fraction of total steps for cosine decay phase, NOT weight decay. `decay=0.0` causes crash; need `decay = 1.0 - warmup_fraction` for standard cosine-with-warmup.
   - **Chat template loss masking**: Both Qwen3 and Qwen2.5 templates use `{% generation %}` / `{% endgeneration %}` tags for SFT loss masking (train only on assistant outputs)
   - **SimpleSFTConfig**: Wrapper config that maps to Levanter trainer config. Key fields: `warmup` (fraction), `decay` (fraction), `weight_decay`, `learning_rate`, `lr_schedule`
   - **compute_per_device_parallelism**: `microbatch_size // num_devices`. For v5p-32: `16 // 16 = 1`
   - **Dataset configs**: `experiments/posttrain/instruction_datasets.py` stores HF dataset IDs with pinned revision hashes
