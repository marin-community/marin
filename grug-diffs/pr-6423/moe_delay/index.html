4"""Token-overhead (iso-loss) analysis for the delayed-gradient / PP cohort.
5 
6The headline PP-viability question is not "how big is the loss gap at a fixed
7step budget" but "how many *extra tokens* does a delayed (pipeline-parallel) run
8need to reach the same loss as synchronous training" -- and whether the gap is a
9recoverable token tax or a permanent quality floor.
10 
11For each run in a W&B group this pulls the train-loss-vs-step history, takes one
12run as the synchronous reference, and reports:
13 
14  * plateau    -- robust final loss (trailing raw mean; no boundary smoothing)
15  * gap        -- plateau minus the sync run's plateau
16  * slope/1k   -- loss change per 1000 steps over the tail (a still-negative
17                  slope means the run is still descending == token tax, not a
18                  converged quality floor)
19  * overhead   -- token-overhead multiplier: the step at which this run reaches
20                  its own plateau loss, divided by the step at which the sync run
21                  first reached that same loss (tokens = steps * batch for a
22                  fixed batch). This is the "extra tokens to match" number that
23                  feeds the throughput break-even in pp_throughput_model.py.
24 
25Smoothing is edge-safe (a centered rolling mean with ``min_periods``) so the tail
26is not dragged toward zero -- a plain ``np.convolve(mode="same")`` pads with

52def _curve(run) -> tuple[np.ndarray, np.ndarray]:
53    """Return (steps, edge-safe-smoothed loss) for a run, sorted by step."""
54    hist = run.history(keys=["train/loss", "_step"], samples=20000)
55    if hist is None or "train/loss" not in getattr(hist, "columns", []):
56        return np.array([]), np.array([])
57    df = hist.dropna(subset=["train/loss"]).sort_values("_step")
58    steps = df["_step"].to_numpy(dtype=float)
59    raw = df["train/loss"].to_numpy(dtype=float)
60    # Centered rolling mean with min_periods keeps the boundaries honest (unlike a
61    # zero-padded convolution, which pulls the final-loss region toward zero).
62    loss = pd.Series(raw).rolling(41, center=True, min_periods=5).mean().to_numpy()
63    return steps, loss

4"""Delayed-gradient optimizer wrappers for studying pipeline-parallel staleness.
5 
6Pipeline-parallel training with a throughput-optimal async schedule applies a
7gradient that was computed ``tau`` weight-versions ago. We study that regime
8*without building pipeline parallelism* by injecting a controlled gradient delay
9in software: the wrapper keeps a depth-``tau`` FIFO of past gradients (and the
10weights they were computed at) inside the optimizer state, and feeds the inner
11optimizer the *stale* gradient each step. This exactly reproduces constant-delay
12asynchronous SGD.
13 
14The FIFO and any correction statistics live in ``opt_state`` — they are the
15"O(weights) extra optimizer state" we are budgeting for — so the canonical grug
16train loop needs no changes; we only swap the optimizer config. At ``tau == 0``
17the wrapper is a pass-through and is bit-identical to the inner optimizer.
18 
19Correctors (applied to the stale gradient before the inner optimizer sees it):
20 
21- ``none``        — naive async SGD; apply the stale gradient as-is.
22- ``dc_asgd``     — DC-ASGD delay compensation (Zheng et al. 2017):
23                    ``g + lambda * (g (.) g) (.) (w_t - w_stale)`` using the
24                    instantaneous squared stale gradient as the diagonal-Hessian
25                    proxy.
26- ``dc_asgd_ema`` — the same correction but with the diagonal curvature read from
27                    an EMA of the squared gradient (i.e. Adam/RMSProp's second
28                    moment ``v_t``). This tests the "reuse the preconditioner
29                    state as the curvature term, near-free" hypothesis.
30 
31For Muon the corrected gradient is fed *before* Newton-Schulz orthogonalization,
32so the correction acts on the momentum/direction and orthogonalization then
33renormalizes the magnitude.
34 
35``weight_pred`` is different in kind: it is a *forward-side* corrector, not a
36gradient correction. Instead of patching the stale gradient, it asks the train
37step to evaluate the gradient at *predicted* weights ``W_hat = w - tau*lr*dW``
38(extrapolating the most recent applied update ``dW`` forward by ``tau`` steps) so
39that, once the gradient is delayed and applied, it lands on the weights it was
40meant for. For Muon ``dW`` is the orthogonalized update (post-orthogonalization
41prediction). The optimizer cannot do this alone — it exposes the predicted offset
42via :meth:`make_forward_predictor` and the grug train step computes the forward
43there; see ``experiments/grug/moe/train.py``.
44 
45Muon-specific forward predictors (``wp_*``) — a pipeline-parallel Muon variant.
46Newton-Schulz decouples the update magnitude from the gradient and makes the
47orthogonalized *direction* noisy step-to-step, so the post-orthogonalization
48``weight_pred`` extrapolates a jittery signal. These predict instead along the
49*smoothed raw momentum* ``m_raw`` (an EMA of the stale gradient, maintained in
50the wrapper — pre-orthogonalization), pointed in the descent direction ``-m_raw``
51and rescaled to the realized per-leaf step RMS:
52 
53- ``wp_preorth``    — raw-momentum-direction prediction (the base PP-Muon mode).
54- ``wp_cautious``   — ``wp_preorth`` with a Cautious-Optimizer sign gate: zero the
55                      coordinates where the predicted descent ``-m_raw`` disagrees
56                      with the realized update, rescaling surviving mass.
57- ``wp_trust``      — ``wp_preorth`` with a LARS-style per-leaf trust-ratio clamp:
58                      cap the offset RMS at ``trust * rms(param)`` (scale-free,
59                      since Muon's magnitude is normalized).
60- ``wp_confidence`` — ``wp_preorth`` scaled by ``relu(cos(-m_raw, last_update))``:
61                      a soft agreement gate that shrinks the prediction when the
62                      momentum and the realized update directions diverge.

108    projection group). A 1F1B/async pipeline applies a gradient ``(P-1-stage)``
109    weight-versions late, so the last stage is fresh (τ=0) and τ increases toward
110    the first stage. ``num_stages == num_layers`` gives one stage per layer

152def wrap_delayed(
153    inner: optax.GradientTransformation,
154    *,
155    tau: int,
156    corrector: str = "none",
157    dc_lambda: float = 1.0,

163    """Wrap ``inner`` so it receives gradients delayed per parameter.

167    each parameter leaf is delayed by ``leaf_tau(path)`` steps — the realistic
168    pipeline-parallel profile where the last stage is fresh (τ=0) and τ grows
169    toward the first stage. The FIFO depth is the maximum τ over the tree; each
170    leaf reads the gradient from its own τ steps ago.
171 
172    Args:
173        inner: the optimizer to feed stale (optionally corrected) gradients to.
174        tau: uniform gradient delay in steps when ``leaf_tau`` is None. ``0`` is a
175            pass-through.

180            staleness damping); ``<1`` shrinks the applied update.
181        pred_beta: EMA decay for the raw-momentum buffer ``m_raw`` that the
182            ``wp_*`` forward predictors extrapolate. Approximates Muon's internal
183            momentum without reaching into the inner optimizer's opaque state.
184        leaf_tau: optional per-leaf delay map (path -> τ) for the per-stage PP
185            profile; overrides ``tau`` when provided.
186    """

227    def _correct(g_stale, v_ema, params, w_stale):
228        if corrector not in ("dc_asgd", "dc_asgd_ema"):
229            # none / lr_damp / forward-predictor modes leave the gradient as-is.
230            return g_stale, v_ema

232        if corrector == "dc_asgd":

236        new_v = jax.tree.map(lambda v, g: dc_beta2 * v + (1.0 - dc_beta2) * (g * g), v_ema, g_stale)
237        corrected = jax.tree.map(lambda g, v, dw: g + dc_lambda * v * dw, g_stale, new_v, delta_w)
238        return corrected, new_v

270def _preorth_leaf_offset(
271    corrector: str,
272    horizon: float,
273    last_update: jax.Array,
274    m_raw: jax.Array,
275    param: jax.Array,
276    trust: float,
277) -> jax.Array:
278    """Predicted-weight offset for one leaf under a ``wp_*`` (pre-orth) corrector.
279 
280    The base prediction points ``horizon`` steps along the *descent* direction of
281    the smoothed raw momentum ``-m_raw``, rescaled to the realized update RMS so
282    its magnitude matches Muon's spectrally-normalized step (which ``last_update``
283    carries) rather than the raw-gradient scale. The ``wp_cautious`` /
284    ``wp_confidence`` / ``wp_trust`` variants then gate or clamp that offset.
285    """
286    descent = -m_raw / _rms(m_raw)  # unit-RMS descent direction (pre-orthogonalization)
287    base = horizon * descent * _rms(last_update)
288    if corrector == "wp_preorth":
289        return base
290    if corrector == "wp_cautious":
291        # Cautious gate: keep coords where the predicted descent agrees with the
292        # realized update, rescale surviving mass to preserve the step size.
293        keep = (descent * last_update > 0).astype(base.dtype)
294        return base * keep / (jnp.mean(keep) + _EPS)
295    if corrector == "wp_confidence":
296        # Soft gate: shrink by the (clipped) cosine between the predicted descent
297        # and the realized update direction. Use sum-of-products, not jnp.vdot:
298        # vdot ravels its operands, and flattening a model-sharded leaf (e.g. the
299        # token embedding) is an unsupported resharding reshape. ``sum`` and the
300        # ``_rms`` reductions are sharding-safe.
301        dot = jnp.sum(descent * last_update)
302        cos = dot / (_rms(descent) * _rms(last_update) * descent.size)
303        return base * jnp.maximum(0.0, cos)
304    if corrector == "wp_trust":
305        # LARS-style clamp: cap the offset RMS at a trust fraction of the param
306        # scale (scale-free, since Muon's update magnitude is normalized).
307        cap = trust * _rms(param)
308        return base * jnp.minimum(1.0, cap / _rms(base))
309    raise ValueError(f"not a pre-orth corrector: {corrector!r}")

4"""Parametric v6e/DCN throughput model for pipeline parallelism vs synchronous DP.
5 
6The staleness experiments answer "what does PP cost in *tokens* (the token
7overhead to reach a target loss)". This answers the other half: "what does PP buy
8in *throughput*", so the two can be combined into a viability verdict:
9 
10    PP is worth it  <=>  throughput_speedup(PP over sync)  >  token_overhead(PP)
11 
12Core mechanism. PP does not change the arithmetic (same FLOPs/token); it changes
13*communication* and *memory*. The decisive quantity for large models trained over
14multiple slices is cross-slice (DCN) traffic:
15 
16    synchronous data/replica parallelism  -> all-reduce the GRADIENTS over DCN:
17        volume ~ N_params  (huge for a large MoE; every expert param is reduced)
18    pipeline parallelism over DCN         -> pass only stage-boundary ACTIVATIONS:
19        volume ~ batch * seq * hidden     (independent of depth / param count)
20 
21So PP's DCN advantage is ~ N_params / (batch*seq*hidden), which *grows with model
22size*. That is the "PP for large MoEs" argument, quantified here.
23 
24Synchronous PP schedules (Zero-Bubble, 1F1B/PipeDream-Flush) are bit-exact and pay
25only a pipeline *bubble*; asynchronous PP removes the bubble but pays the measured
26token overhead. Both are modeled.
27 
28All hardware numbers are explicit, conservative, and easy to override -- the goal
29is the *regime* and *break-even*, not false precision.
30 
31    .venv/bin/python -m experiments.grug.moe_delay.pp_throughput_model
32"""

248def _make_train_step(
249    optimizer: optax.GradientTransformation,
250    mp: jmp.Policy,
251    *,
252    z_loss_weight: float,
253    ema_beta: float | None,
254    watch_config: WatchConfig | None = None,
255):
256    one = jnp.array(1, dtype=jnp.int32)
257    z_loss = z_loss_weight if z_loss_weight > 0 else None
258    if watch_config is not None:
259        if isinstance(watch_config.watch_targets, str):
260            watch_targets = tuple(t.strip() for t in watch_config.watch_targets.split(","))
261        else:
262            watch_targets = tuple(watch_config.watch_targets)
263    else:
264        watch_targets = ()
265 
266    @functools.partial(jax.jit, donate_argnums=(0,), static_argnames=("compute_watch",))
267    def train_step(state: GrugTrainState, batch, *, compute_watch: bool = False):
268        def loss_fn(params):
269            compute_params = mp.cast_to_compute(params)
270            return compute_params.next_token_loss(
271                batch.tokens,
272                batch.loss_weight,
273                mask=batch.attn_mask,
274                reduction="mean",
275                logsumexp_weight=z_loss,
276            )
277 
278        loss, grads = jax.value_and_grad(loss_fn)(state.params)
279        updates, opt_state = optimizer.update(grads, state.opt_state, state.params)
280        params = optax.apply_updates(state.params, updates)
281 
282        if ema_beta is None:
283            ema_params = None
284        else:
285            if state.ema_params is None:
286                raise ValueError("ema_params must be initialized when ema_beta is set.")
287            ema_params = jax.tree_util.tree_map(
288                lambda old, new: ema_beta * old + (1.0 - ema_beta) * new,
289                state.ema_params,
290                params,
291            )
292 
293        watch_stats = None
294        if watch_config is not None and compute_watch:
295            watch_stats = compute_watch_stats(
296                watch_targets=watch_targets,
297                include_norms=watch_config.include_norms,
298                include_per_parameter_norms=watch_config.include_per_parameter_norms,
299                include_histogram=watch_config.include_histograms,
300                split_scan_layers=watch_config.split_scan_layers,
301                params=state.params,
302                grads=grads,
303                updates=updates,
304                opt_state=state.opt_state,
305                model_tree_type=type(state.params),
306            )
307 
308        next_state = dataclasses.replace(
309            state,
310            step=state.step + one,
311            params=params,
312            opt_state=opt_state,
313            ema_params=ema_params,
314        )
315 
316        return next_state, {"train/loss": loss}, watch_stats
317 
318    return train_step

322    """Run grug training locally inside a Fray worker."""
323    trainer = config.trainer.trainer
324    trainer.initialize()
325    levanter.tracker.log_configuration(config)
326 
327    run_id = trainer.id
328    if run_id is None:
329        raise ValueError("trainer.id was not initialized")
330 
331    optimizer = config.optimizer.build(trainer.num_train_steps)
332    watch_config = trainer.watch

354            batch_schedule=batch_schedule,
355            key=data_key,
356        )

358            train_dataset,
359            batch_schedule=batch_schedule,
360            mesh=mesh,
361            batch_pspec=config.trainer.train_batch_pspec,
362        )
363 
364        @jax.jit
365        def _init_state(model_rng):
366            return initial_state(
367                config.model,
368                optimizer=optimizer,
369                mp=trainer.mp,
370                key=model_rng,
371                ema_beta=config.trainer.ema_beta,
372            )
373 
374        state = _init_state(model_key)
375 
376        checkpointer = trainer.checkpointer.create(run_id)

405        iterator = LoadingTimeTrackerIterator(train_loader.iter_from_step(int(state.step)))

408            step_getter=lambda s: s.step,
409            model_getter=lambda s: s.params,
410            eval_model_getter=lambda s: s.ema_params if s.ema_params is not None else s.params,
411            opt_state_getter=lambda s: s.opt_state,

415            every=log_every,
416        )
417        state_callbacks.add_hook(callbacks.pbar_logger(total=trainer.num_train_steps), every=log_every)
418        state_callbacks.add_hook(callbacks.log_step_info(trainer.num_train_steps), every=log_every)
419        if profiler_enabled:
420            state_callbacks.add_hook(
421                callbacks.profile(
422                    str(trainer.log_dir / run_id / "profiler"),
423                    profiler_cfg.start_step,
424                    profiler_num_steps,
425                    profiler_cfg.perfetto_link,
426                ),
427                every=1,
428            )
429        state_callbacks.add_hook(_make_mixture_stage_callback(train_dataset, batch_schedule), every=1)
430        if evaluator is not None and eval_cfg is not None:
431            interval = eval_cfg.steps_per_eval
432            eval_ema = eval_cfg.eval_ema and config.trainer.ema_beta is not None
433            if interval is not None and interval > 0 and (eval_cfg.eval_current or eval_ema):
434                state_callbacks.add_hook(
435                    cb_tagged_evaluate(
436                        evaluator,
437                        prefix=eval_cfg.prefix,
438                        eval_current=eval_cfg.eval_current,
439                        eval_ema=eval_ema,
440                    ),
441                    every=interval,
442                )
443 
444        last_loss: float | jax.Array = 0.0
445        last_step_duration = 0.0
446 
447        # Main optimization loop.
448        try:
449            while int(state.step) < trainer.num_train_steps:
450                with jax.profiler.TraceAnnotation("load_batch"):
451                    batch = next(iterator)
452                step_start = time.perf_counter()
453                current_step = int(state.step)
454                # grad_watch runs only on its configured interval.
455                compute_watch = (
456                    watch_config.is_enabled and watch_config.interval > 0 and current_step % watch_config.interval == 0
457                )
458                state, metrics, watch_stats = train_step(state, batch, compute_watch=compute_watch)
459                step = int(state.step) - 1
460 
461                jax.block_until_ready(metrics["train/loss"])
462                duration = time.perf_counter() - step_start
463                hook_start = time.perf_counter()
464                with jax.profiler.TraceAnnotation("callbacks"):
465                    state_callbacks.run(state, loss=metrics["train/loss"], step_duration=duration)
466                    last_loss = metrics["train/loss"]
467                    last_step_duration = duration
468                    levanter.tracker.log({"throughput/hook_time": time.perf_counter() - hook_start}, step=step)
469                    levanter.tracker.log({"throughput/loading_time": iterator.this_load_time}, step=step)
470 
471                    if watch_stats is not None:
472                        levanter.tracker.log(watch_stats, step=step)
473 
474                if checkpointer is not None:
475                    checkpointer.on_step(tree=state, step=int(state.step))
476        except BaseException:
477            logger.exception(
478                "Fatal error in grug training loop; skipping final callbacks/checkpoint to preserve root cause"
479            )
480            raise
481        else:
482            # Mirror classic trainer behavior: force callbacks on the last completed step.
483            state_callbacks.run(state, loss=last_loss, step_duration=last_step_duration, force=True)
484            if checkpointer is not None:
485                checkpointer.on_step(tree=state, step=int(state.step), force=True)
486                checkpointer.wait_until_finished()
487 
488    levanter.tracker.current_tracker().finish()
