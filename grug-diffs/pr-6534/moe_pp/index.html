tt1# moe_pp — pipeline parallelism for the production grug-MoE
2 
3Goal: pipeline parallelism (toward zero-bubble) for the **production** grug-MoE,
4composing with expert parallelism (EP) and FSDP, on both TPU and GPU. PP is the
5final parallelism axis added on top of the existing EP/FSDP grug stack.
6 
7## What's here (the TPU baseline)
8 
9`pipeline.py` runs the production `Transformer` inside a single `shard_map` that
10manualizes all mesh axes, with a per-stage manual `jax.vjp` GPipe backward,
11inline ring-EP, the megablox GMM, and a manual FSDP weight all-gather. It is
12gradient-exact vs the non-pipelined oracle and trains on real hardware
13(v6e-8 ~0.78× FSDP single-host; v6e-32 ~1.22× FSDP — PP wins once the param
14all-gather crosses DCN). `benchmark.py` drives it; `oracle.py` is the
15non-pipelined reference.
16 
17## What we tried and dropped
18 
19- **`stage`-axis `shard_map` + GSPMD for data/expert** (the `moe_zb` toy). The
20  schedule and the pairwise PP×FSDP / PP×EP compositions worked, but the full
21  PP×FSDP×EP does not lower on TPU: XLA's SPMD partitioner can't factor two
22  GSPMD axes' device groups under a third manual axis. Fix was to manualize all
23  axes (above), so no GSPMD axis touches any operand.
24- **Autodiff through the forward pipeline** as the backward. It materializes a
25  weight-grad buffer stacked across all stages (~48 GiB at 40B) that OOMs the
26  XLA partitioner. Replaced by the per-stage manual `jax.vjp` backward, whose
27  cotangents never carry a `[num_stages, …]` axis.
28- **TPU Pallas remote-DMA ring-shift** (a drop-in for `ppermute`). De-risked
29  and bit-exact vs `ppermute`, but unnecessary: XLA sharding on TPU is fine, so
30  TPU needs no custom kernel. Dropped.
31 
32## Direction
33 
34- **PP is threaded manually** (explicit per-round communication / separate
35  launches), not via a `stage` mesh axis.
36- **EP/FSDP** stay XLA / jax map operations on **TPU**, but must be expressed as
37  **Pallas kernels on GPU** — XLA cannot lower the combined PP×EP×FSDP sharding
38  on GPU and OOMs. The grug refactor makes EP/FSDP expressible either way:
39  Pallas on GPU, traditional jax map operations on TPU.

171def ep_pipeline_mesh(*, stage: int, expert: int, replica: int, data: int, model: int = 1) -> Mesh:
172    """Mesh for the inline ring-EP pipeline: ALL axes Explicit (fully manualized).
173 
174    Every axis is ``AxisType.Explicit`` so the outer shard_map can manualize all five.
175    The load-bearing axes are ``stage`` (pipeline), ``expert`` (EP), and ``data`` (the
176    dispatched tokens are batch-sharded over it); ``replica_dcn`` / ``model`` are size-1
177    but still manualized. This is required on TPU: the megablox GMM (``ragged_dot`` ->
178    ``_gmm_megablox``) is a Mosaic/Pallas kernel GSPMD cannot auto-partition, and the
179    SPMD partitioner processes every GSPMD axis for every op (including size-1 ones the
180    operands are merely replicated over), so even one residual Auto axis trips the
181    Mosaic auto-partition error. Manualizing all five leaves zero GSPMD axis on the
182    kernel; with no live Explicit axis inside, the xla ragged_dot path also lowers.
183    """
184    shape = (stage, replica, data, expert, model)
185    if int(np.prod(shape)) != jax.device_count():
186        raise ValueError(f"mesh shape {shape} (prod={int(np.prod(shape))}) must use all {jax.device_count()} devices")
187    devices = np.array(jax.devices(), dtype=object).reshape(shape)
188    # ALL axes Explicit so the outer shard_map can manualize EVERY axis. The megablox
189    # GMM (a Mosaic kernel) cannot survive ANY GSPMD partition attempt -- the XLA SPMD
190    # partitioner processes every GSPMD axis for every op, including size-1 ones the
191    # operands are merely replicated over, so a single residual Auto axis is enough to
192    # trip "Mosaic kernels cannot be automatically partitioned." Manualizing all five
193    # leaves zero GSPMD axis touching the kernel. With no live Explicit axis inside the
194    # manual region the xla ragged_dot path lowers too (no Explicit-axis sharding-rule
195    # error), so this mesh works under both RAGGED_DOT_IMPL=xla and megablox.
196    axis_types = (AxisType.Explicit,) * 5
197    return Mesh(devices, _GRUG_MESH_AXIS_NAMES, axis_types=axis_types)

364def _stage_in_specs(stage_block_arrays: eqx.Module) -> eqx.Module:
365    """In-specs for the stacked block arrays: ZeRO-3 FSDP over ``data`` + EP over ``expert``.
366 
367    Every block leaf is sharded over ``stage`` (leading dim). On top of that:
368 
369    - The expert MLP weights ``mlp.expert_mlp.w_gate_up`` ``[stage, layers, E, D, I2]``
370      and ``...w_down`` ``[stage, layers, E, I, D]`` shard their expert dim (array axis
371      2) over the manual ``expert`` axis (so each shard holds ``E / expert_size``
372      experts, as :func:`_moe_mlp_ep_ring_local` expects) AND shard a feature dim over
373      ``data`` for FSDP -- ``I2`` (axis 4) of ``w_gate_up`` and ``I`` (axis 3) of
374      ``w_down``.
375    - Every other large block weight (attention projections, the shared dense MLP, the
376      router, and the gated-norm factor matrices) shards its largest feature dim over
377      ``data``.
378 
379    The body :func:`_fsdp_all_gather` reconstructs each weight over ``data`` before use;
380    the autodiff transpose turns that all-gather into a reduce-scatter, so the returned
381    weight grads (hence the optimizer state) stay sharded ``/data`` -- true ZeRO-3.
382    Per-device weight+optimizer memory then scales as ``total/(stage*expert*data)``.
383    """
384    default_spec = P(STAGE_AXIS)
385    specs = jax.tree_util.tree_map(lambda _: default_spec, stage_block_arrays)
386 
387    # Stacked arrays carry a leading [stage, layers_per_stage, ...]; the feature axis to
388    # FSDP-shard over ``data`` is named by absolute position in the stacked array.
389    def _set(specs_tree, accessor, spec):
390        return eqx.tree_at(accessor, specs_tree, spec, is_leaf=lambda x: x is None)
391 
392    # Expert MLP: expert dim (axis 2) over ``expert``; a non-expert feature dim over ``data``.
393    expert_mlp = specs.mlp.expert_mlp
394    expert_mlp = eqx.tree_at(
395        lambda m: (m.w_gate_up, m.w_down),
396        expert_mlp,
397        (
398            P(STAGE_AXIS, None, EXPERT_AXIS, None, DATA_AXIS),  # w_gate_up [stage,layers,E,D,I2] -> shard I2
399            P(STAGE_AXIS, None, EXPERT_AXIS, DATA_AXIS, None),  # w_down    [stage,layers,E,I,D] -> shard I
400        ),
401    )
402    specs = eqx.tree_at(lambda s: s.mlp.expert_mlp, specs, expert_mlp)
403 
404    # Router [stage,layers,D,E] -> shard D (axis 2); router_bias [stage,layers,E] stays stage-only (tiny).
405    specs = _set(specs, lambda s: s.mlp.router, P(STAGE_AXIS, None, DATA_AXIS, None))
406 
407    # Attention projections [stage,layers,*,*] -> shard the output feature dim over ``data``.
408    specs = _set(specs, lambda s: s.attn.w_q, P(STAGE_AXIS, None, None, DATA_AXIS))
409    specs = _set(specs, lambda s: s.attn.w_k, P(STAGE_AXIS, None, None, DATA_AXIS))
410    specs = _set(specs, lambda s: s.attn.w_v, P(STAGE_AXIS, None, None, DATA_AXIS))
411    specs = _set(specs, lambda s: s.attn.w_o, P(STAGE_AXIS, None, None, DATA_AXIS))
412    specs = _set(specs, lambda s: s.attn.attn_gate, P(STAGE_AXIS, None, DATA_AXIS, None))
413 
414    # Gated norms (attn + mlp): w_down [stage,layers,D,R] -> shard D; w_up [stage,layers,R,D] -> shard D.
415    for gn in (lambda s: s.attn_gated_norm, lambda s: s.mlp_gated_norm):
416        specs = _set(specs, lambda s, _gn=gn: _gn(s).w_down, P(STAGE_AXIS, None, DATA_AXIS, None))
417        specs = _set(specs, lambda s, _gn=gn: _gn(s).w_up, P(STAGE_AXIS, None, None, DATA_AXIS))
418 
419    # Shared dense MLP: all three matrices shard the shared-intermediate dim over ``data``.
420    if specs.shared is not None:
421        specs = _set(specs, lambda s: s.shared.w_gate, P(STAGE_AXIS, None, None, DATA_AXIS))
422        specs = _set(specs, lambda s: s.shared.w_up, P(STAGE_AXIS, None, None, DATA_AXIS))
423        specs = _set(specs, lambda s: s.shared.w_down, P(STAGE_AXIS, None, DATA_AXIS, None))
424 
425    return specs

515def _constrain_to_specs(grads: eqx.Module, specs: eqx.Module, mesh: jax.sharding.Mesh) -> eqx.Module:
516    """Reshard each grad leaf to its param's in-spec on the SAME run mesh.
517 
518    ``jax.value_and_grad`` of the stage-manual ``shard_map`` returns the weight grads
519    stage-REPLICATED -- the transpose of the stage-sharded forward produces a
520    ``P(None, ...)`` cotangent on the stage axis even though the param is
521    ``P("stage", ...)``. Left as-is, the partitioner only discovers the mismatch when
522    optax combines the grad with the stage-sharded param / opt-state, and reshards
523    stage-replicated -> stage-sharded by involuntary full rematerialization: it
524    materializes the entire stacked weight grad un-sharded on one device. Resharding
525    each grad leaf to the param's ``NamedSharding(mesh, spec)`` here -- the same mesh,
526    so it is an intra-mesh slice, never a cross-``Mesh`` materialization -- pins the
527    target sharding at grad-production time so the grad exits already stage-sharded and
528    optax sees matching shardings. ``reshard`` (not ``with_sharding_constraint``) is
529    required for the Explicit axes: ``with_sharding_constraint`` acts as an assert that the
530    array is ALREADY in the spec, which the replicated grad is not. Any ``AxisType.Auto``
531    axis cannot be named by ``reshard``, so its projection is constrained separately with
532    ``with_sharding_constraint``; on the all-Explicit pipeline mesh there are no Auto axes,
533    so only the ``reshard`` path runs.
534    """
535    auto_axes = frozenset(name for name, at in zip(mesh.axis_names, mesh.axis_types, strict=True) if at == AxisType.Auto
 >)
536 
537    def _constrain(g, spec):
538        explicit = P(*(None if name in auto_axes else name for name in spec))
539        g = reshard(g, NamedSharding(mesh, explicit))
540        if any(name in auto_axes for name in spec):
541            auto = P(*(name if name in auto_axes else None for name in spec))
542            g = jax.lax.with_sharding_constraint(g, NamedSharding(mesh, auto))
543        return g

548def _ep_cross_entropy(
549    final_hidden: jax.Array,
550    output_proj: jax.Array,
551    labels: jax.Array,
552    weight: jax.Array,
553    *,
554    batch_axes: tuple[str, ...],
555) -> jax.Array:
556    """Mean next-token cross-entropy whose token reduction spans the batch-sharding axes.
557 
558    ``final_hidden`` is this shard's local token slice. The cross-entropy is a weighted
559    mean over ALL tokens; sum the per-shard NLL and weight, ``psum`` each over the MANUAL
560    batch axes (``batch_axes``), then divide -- so every shard returns the full-batch
561    mean.
562 
563    ``batch_axes`` is ``(expert, data)`` when both are Manual (the TPU all-manual path)
564    and ``(expert,)`` when ``data`` is a GSPMD axis (the GPU path): there the residual
565    ``data`` reduction of the separate NLL / weight sums is inserted by GSPMD when the
566    replicated ``P()`` loss is materialized -- it all-reduces each sum over ``data``
567    before the division, so the global weighted mean is exact (the division is applied
568    to the two already-data-reduced sums, not per-data-group then averaged).
569    """
570    logits = jnp.einsum("bsd,dv->bsv", final_hidden, output_proj).astype(jnp.float32)
571    log_z = jax.scipy.special.logsumexp(logits, axis=-1)
572    label_logit = jnp.take_along_axis(logits, labels[..., None], axis=-1)[..., 0]
573    nll = jnp.sum((log_z - label_logit) * weight)
574    wsum = jnp.sum(weight)
575    nll = jax.lax.psum(nll, batch_axes)
576    wsum = jax.lax.psum(wsum, batch_axes)
577    return nll / wsum

580# --- Microbatched (GPipe-scheduled) inline ring-EP pipeline ---------------------
581#
582# The loss below splits the batch into ``num_microbatches`` microbatches and runs the
583# GPipe schedule: stage 0 injects microbatch ``tick`` at timestep ``tick``, every stage
584# runs its blocks on its current buffer each tick, and the activation is ``ppermute``d
585# downstream stage->stage+1. The last stage collects each microbatch's drained hidden
586# and the head scores them in one fused pass after the sweep, so the bubble shrinks to
587# ``(S-1)/(M+S-1)``.
588#
589# The body is differentiated by whole-program ``jax.value_and_grad`` (no manual
590# backward): the forward ppermute transposes to a reverse ppermute and the inline
591# ring's all_gather/psum_scatter transpose cleanly because ``expert`` and ``data`` are
592# both manual (no live GSPMD axis under the manual region).
593

595def _ep_pipeline_loss_microbatched(
596    embed_arrays: eqx.Module,
597    stage_block_arrays: eqx.Module,
598    embed_static: eqx.Module,
599    block_static: eqx.Module,
600    transformer: Transformer,
601    token_microbatches: jax.Array,
602    weight_microbatches: jax.Array,
603    *,
604    mesh: jax.sharding.Mesh,
605    num_stages: int,
606    num_microbatches: int,
607) -> jax.Array:
608    """Microbatched (GPipe) next-token loss with the real ring EP inline; ``{stage, expert, data}`` manual.
609 
610    Runs the GPipe microbatch schedule inside the single ``{stage, expert, data}``-manual
611    ``shard_map``. ``token_microbatches`` / ``weight_microbatches`` are
612    ``[num_microbatches, microbatch, seq]``; each microbatch's tokens shard their
613    per-microbatch batch dim over BOTH the manual ``expert`` and ``data`` axes
614    (``P(None, (EXPERT_AXIS, DATA_AXIS), None)``). The loss is the full-batch weighted
615    next-token CE (one fused score over all microbatches' drained hiddens) plus the
616    router z-aux averaged over microbatches.
617    """
618    num_layers = len(transformer.blocks)
619    cfg = transformer.config
620    seq_len = token_microbatches.shape[-1]
621 
622    layer_masks = build_layer_masks(transformer, num_stages, seq_len)
623 
624    stage_spec = P(STAGE_AXIS)
625    stage_in_specs = _stage_in_specs(stage_block_arrays)
626    embed_in_specs = _embed_in_specs(embed_arrays)
627    # Each microbatch's tokens shard their per-microbatch batch axis over both expert
628    # and data; the leading num_microbatches axis is replicated.
629    token_spec = P(None, _EP_BATCH_AXES, None)
630 
631    def body(stage_arrays, embed, masks, tokens, weights):
632        sid = jax.lax.axis_index(STAGE_AXIS)
633        S = num_stages
634        M = num_microbatches
635        T = M + S - 1
636        fwd_perm = [(i, i + 1) for i in range(S - 1)]
637        is_first = sid == 0
638        is_last = sid == (S - 1)
639 
640        # FSDP: all-gather every ``data``-sharded weight over ``data`` before use; the
641        # transpose reduce-scatters the grads back ``/data``.
642        stage_arrays = _fsdp_all_gather(stage_arrays, stage_in_specs)
643        embed = _fsdp_all_gather(embed, embed_in_specs)
644 
645        token_embed, embed_norm, embed_gated_norm, final_norm, final_gated_norm, output_proj = eqx.combine(
646            embed, embed_static
647        )
648        stage_blocks = jax.tree_util.tree_map(lambda x: x[0], stage_arrays)
649        stage_masks = masks[0]
650 
651        # Activation buffer for one microbatch's local token slice. ``expert`` and
652        # ``data`` are both manual, so ``tokens`` is already this shard's slice -- the
653        # local per-microbatch row count is ``tokens.shape[1]`` (= microbatch /
654        # (expert_size * data_size)).
655        local_microbatch = tokens.shape[1]
656        hidden_shape = (local_microbatch, seq_len, cfg.hidden_dim)
657        buf = jnp.zeros(hidden_shape, jnp.float32)
658        z_total = jnp.zeros((), jnp.float32)
659        # Last stage drains microbatch m into h_final[m]; every other stage / invalid
660        # slot adds zero, so each microbatch is written exactly once.
661        h_final = jnp.zeros((M, *hidden_shape), jnp.float32)
662 
663        # Remat the per-stage block-scan: across the T = M+S-1 schedule steps,
664        # whole-program value_and_grad would otherwise save every (step x layer) forward
665        # (~T * layers_per_stage activations). Recompute each step's stage forward in the
666        # backward so activation memory scales with one stage, not the whole schedule.
667        run_stage = eqx.filter_checkpoint(_run_stage_blocks)
668        for t in range(T):
669            m = t - sid
670            valid = (m >= 0) & (m < M)
671            m_clip = jnp.clip(m, 0, M - 1)
672            tok_m = jax.lax.dynamic_index_in_dim(tokens, m_clip, axis=0, keepdims=False)
673 
674            embedded = token_embed[tok_m]
675            embedded = embed_gated_norm(embed_norm(embedded))
676            stage_in = jnp.where(is_first, embedded, buf)
677 
678            stage_out, z_local = run_stage(stage_blocks, block_static, stage_in, stage_masks)
679            # Per-stage router z-loss is local to this stage's layers; count it once
680            # per microbatch (every valid slot processes a distinct microbatch).
681            z_total = z_total + jnp.where(valid, z_local, 0.0)
682 
683            contrib = jnp.where(is_last & valid, stage_out, jnp.zeros_like(stage_out))
684            prev = jax.lax.dynamic_index_in_dim(h_final, m_clip, axis=0, keepdims=False)
685            h_final = jax.lax.dynamic_update_index_in_dim(h_final, prev + contrib, m_clip, axis=0)
686 
687            buf = jax.lax.ppermute(stage_out, STAGE_AXIS, fwd_perm)
688 
689        # Replicate each microbatch's last-stage hidden onto every stage (others held
690        # zero), then score the whole global batch in ONE fused CE -- a single weighted
691        # mean over all microbatches' tokens, identical to the non-pipelined oracle.
692        h_final = jax.lax.psum(h_final, STAGE_AXIS)
693        flat_batch = M * local_microbatch
694        final_hidden = final_gated_norm(final_norm(h_final.reshape(flat_batch, seq_len, cfg.hidden_dim)))
695        tokens_flat = tokens.reshape(flat_batch, seq_len)
696        weights_flat = weights.reshape(flat_batch, seq_len)
697        labels = _next_token_labels(tokens_flat)
698        ce = _ep_cross_entropy(
699            final_hidden, output_proj, labels, weights_flat.astype(jnp.float32), batch_axes=_EP_BATCH_AXES
700        )
701 
702        # Router z-loss: per-(stage,microbatch) it is a token mean over this
703        # (expert, data) shard's slice. The full-batch mean is the mean over stages'
704        # layers, (expert, data) shards, AND microbatches -- so sum over stages, average
705        # over both manual batch axes, and divide by both num_layers and num_microbatches.
706        z_total = jax.lax.psum(z_total, STAGE_AXIS)
707        z_total = jax.lax.psum(z_total, _EP_BATCH_AXES) / jax.lax.psum(1, _EP_BATCH_AXES)
708        aux = cfg.router_z_loss_coef * (z_total / num_layers / num_microbatches)
709        return ce + aux
710 
711    def _place(x, spec):
712        return reshard(x, NamedSharding(mesh, spec))
713 
714    stage_block_arrays = jax.tree_util.tree_map(_place, stage_block_arrays, stage_in_specs)
715    embed_arrays = jax.tree_util.tree_map(_place, embed_arrays, embed_in_specs)
716    layer_masks = _place(layer_masks, stage_spec)
717    token_microbatches = _place(token_microbatches, token_spec)
718    weight_microbatches = _place(weight_microbatches, token_spec)
719 
720    return shard_map(
721        body,
722        mesh=mesh,
723        in_specs=(stage_in_specs, embed_in_specs, stage_spec, token_spec, token_spec),
724        out_specs=P(),
725        axis_names=frozenset(_GRUG_MESH_AXIS_NAMES),
726        check_vma=False,
727    )(stage_block_arrays, embed_arrays, layer_masks, token_microbatches, weight_microbatches)

268        def loss_fn(params):

270            return compute_params.next_token_loss(
271                batch.tokens,

273                mask=batch.attn_mask,
274                reduction="mean",
275                logsumexp_weight=z_loss,
276            )

279        updates, opt_state = optimizer.update(grads, state.opt_state, state.params)
280        params = optax.apply_updates(state.params, updates)

289                state.ema_params,
290                params,
291            )
292 
293        watch_stats = None
294        if watch_config is not None and compute_watch:

301                params=state.params,

303                updates=updates,

470

479            )

483            state_callbacks.run(state, loss=last_loss, step_duration=last_step_duration, force=True)
484            if checkpointer is not None:
