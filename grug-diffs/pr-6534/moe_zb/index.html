5 
6This is a deliberately legible re-implementation of the grug MoE architecture
7(``experiments/grug/moe/model.py``) with every mesh-coupled construct removed:
8no ``jax.sharding.reshard``, no ``shard_map``, no abstract-mesh lookups. The
9production model's routing path uses ``reshard`` calls whose axes turn ``Manual``

11it can be driven by the zero-bubble pipeline primitive.
12

14

16  no sharded beta, no dynamic router bias).

18  gate-weighted sum over the selected ones — exact, no capacity dropping.
19- SwiGLU expert MLPs (silu(gate) * up, then down).
20- Multi-head attention with rotary embeddings and a manual causal softmax; GQA is
21  supported via key/value head expansion.

4"""Pipeline-parallel primitives for grug, with a zero-bubble schedule.
5 
6This is the pipeline analogue of Levanter's micro-batching
7(`levanter.grad_accum.microbatched`): micro-batching splits the global batch
8into microbatches and folds a forward+backward over them on one set of devices;
9here the same microbatch loop is *distributed across pipeline stages* so each
10stage owns a contiguous slice of the model's layers and the microbatches stream
11through the stages.
12 
13Two backends compute the same gradients:
14 
15- ``pipeline_value_and_grad`` runs the forward pipeline inside a ``shard_map``
16  over a ``stage`` mesh axis (activations shipped stage->stage+1 with
17  ``jax.lax.ppermute``) and lets ``jax.value_and_grad`` build the backward. The
18  reverse-mode transpose of the forward ppermute ships cotangents stage->stage-1,
19  so this is a correct GPipe schedule. It is the reference oracle.
20 
21- ``zero_bubble_value_and_grad`` runs an explicit schedule that splits each
22  stage's backward into B (gradient w.r.t. the stage input, which unblocks the
23  upstream stage) and W (gradient w.r.t. the stage weights, which blocks
24  nothing), and slots the W work into the warmup/cooldown bubbles. It must match
25  the oracle's gradients bit-for-tolerance.
26 
27The model enters as three SPMD-identical callables so a single program runs on
28every stage:
29 
30- ``embed_fn(embed_params, token_microbatch) -> hidden`` (used on stage 0),
31- ``stage_fn(stage_params, hidden_in) -> hidden_out`` (every stage),
32- ``head_loss_fn(head_params, hidden, target_microbatch) -> scalar`` (stage N-1).
33 
34``stage_params`` carries a leading ``layers_per_stage`` axis; ``stage_fn``
35applies that stage's blocks in order. ``embed_params``/``head_params`` are
36replicated across stages (only stages 0 / N-1 consume them) so the single
37program type-checks everywhere.
38"""

91def _pipeline_forward_hidden(
92    params_stage: PyTree,
93    params_embed: PyTree,
94    tokens: jax.Array,
95    *,
96    model: PipelineModel,
97    num_stages: int,
98    num_microbatches: int,
99    hidden_shape: tuple[int, ...],
100    hidden_dtype: jnp.dtype,
101) -> tuple[jax.Array, jax.Array]:
102    """Forward pipeline carrying only hidden states, run *inside* a shard_map.
103 
104    Each device holds one stage. At timestep ``t`` stage ``s`` works on microbatch
105    ``m = t - s``: stage 0 embeds ``tokens[m]`` (a cheap gather, kept here +
106    masked), every other stage consumes the activation ``ppermute`` delivered from
107    stage ``s-1`` last step. The head is *not* here — instead the final stage's
108    valid output for microbatch ``m`` is written into ``h_final[m]`` so a single
109    head can score every microbatch once, outside the per-stage program.
110 
111    Returns ``(h_final, aux_total)`` — ``h_final`` is ``[num_microbatches,
112    *hidden_shape]`` and ``aux_total`` the summed per-stage aux; both are reduced
113    onto every stage (``psum``) so the caller reads a stage-replicated value.
114    """
115    sid = jax.lax.axis_index(STAGE_AXIS)
116    S = num_stages
117    M = num_microbatches
118    T = M + S - 1
119    fwd_perm = [(i, i + 1) for i in range(S - 1)]
120    is_first = sid == 0
121    is_last = sid == (S - 1)
122 
123    stage_params = jax.tree_util.tree_map(lambda x: x[0], params_stage)
124 
125    buf = jnp.zeros(hidden_shape, hidden_dtype)
126    aux_total = jnp.zeros((), jnp.float32)
127    h_final = jnp.zeros((M, *hidden_shape), hidden_dtype)
128    for t in range(T):
129        m = t - sid
130        valid = (m >= 0) & (m < M)
131        m_clip = jnp.clip(m, 0, M - 1)
132        tok_m = jax.lax.dynamic_index_in_dim(tokens, m_clip, axis=0, keepdims=False)
133        embedded = model.embed_fn(params_embed, tok_m)
134        stage_in = jnp.where(is_first, embedded, buf)
135        stage_out, stage_aux = model.stage_fn(stage_params, stage_in)
136        # Per-stage aux (e.g. router z-loss) is local to a stage's own layers and
137        # summed across stages — count it on every valid slot, not just the last.
138        aux_total = aux_total + jnp.where(valid, stage_aux, 0.0)
139        # Last stage's valid output for microbatch m is its final hidden; add it
140        # into slot m (other stages / invalid slots add zero — each m written once).
141        contrib = jnp.where(is_last & valid, stage_out, jnp.zeros_like(stage_out))
142        prev = jax.lax.dynamic_index_in_dim(h_final, m_clip, axis=0, keepdims=False)
143        h_final = jax.lax.dynamic_update_index_in_dim(h_final, prev + contrib, m_clip, axis=0)
144        buf = jax.lax.ppermute(stage_out, STAGE_AXIS, fwd_perm)
145 
146    h_final = jax.lax.psum(h_final, STAGE_AXIS)
147    aux_total = jax.lax.psum(aux_total, STAGE_AXIS)
148    return h_final, aux_total

151def pipeline_value_and_grad(
152    params: PipelineParams,
153    tokens: jax.Array,
154    targets: jax.Array,
155    *,
156    model: PipelineModel,
157    mesh: jax.sharding.Mesh,
158    num_microbatches: int,
159    hidden_shape: tuple[int, ...],
160    hidden_dtype: jnp.dtype = jnp.float32,
161) -> tuple[jax.Array, PipelineParams]:
162    """(loss, grads) with the head hoisted out of the per-stage pipeline.
163 
164    The pipeline ``shard_map`` carries only hidden states; the final-stage hidden
165    of every microbatch is emitted and a *single* head scores them with the batch
166    (``num_microbatches * microbatch``) axis sharded over ``stage``, so the
167    ``[D, V]`` projection runs once across all stages rather than redundantly on
168    each (a per-stage ``lax.cond`` does not help — XLA predicates it). Sequences
169    stay whole on their stage, so the head's next-token shift is intact.
170    ``jax.value_and_grad`` builds the GPipe-schedule backward and the head grad
171    reduces over ``stage``.
172 
173    ``tokens``/``targets`` are ``[num_microbatches, microbatch, seq]``;
174    ``hidden_shape`` is the per-microbatch activation shape ``[microbatch, seq, D]``.
175    """
176    num_stages = mesh.shape[STAGE_AXIS]
177    M = num_microbatches
178    microbatch = hidden_shape[0]
179    batch = M * microbatch
180    if batch % num_stages != 0:
181        raise ValueError(
182            f"num_microbatches*microbatch={batch} must be divisible by num_stages={num_stages} "
183            "to shard the head's batch axis over the stage axis"
184        )
185    stage_spec = P(STAGE_AXIS)
186    repl = P()
187 
188    def full_loss(p: PipelineParams) -> jax.Array:
189        h_final, aux_total = shard_map(
190            lambda ps, pe, tk: _pipeline_forward_hidden(
191                ps,
192                pe,
193                tk,
194                model=model,
195                num_stages=num_stages,
196                num_microbatches=M,
197                hidden_shape=hidden_shape,
198                hidden_dtype=hidden_dtype,
199            ),
200            mesh=mesh,
201            in_specs=(stage_spec, repl, repl),
202            out_specs=(repl, repl),
203            axis_names=frozenset({STAGE_AXIS}),
204        )(p.stage, p.embed, tokens)
205        # Distribute the single head over the stage axis: collapse [M, microbatch]
206        # into one batch axis and shard it, so each stage scores 1/S of the (whole)
207        # sequences. The [D, V] projection and its grad then run once across stages.
208        # Generic over the per-microbatch hidden rank (dense [mb, H] or grug
209        # [mb, seq, D]) — only the leading batch axis is sharded.
210        h_rest = hidden_shape[1:]
211        h_batched = h_final.reshape(batch, *h_rest)
212        h_dist = jax.device_put(h_batched, NamedSharding(mesh, P(STAGE_AXIS, *([None] * len(h_rest)))))
213        tgt_rest = targets.shape[2:]
214        tgt_batched = targets.reshape(batch, *tgt_rest)
215        tgt_dist = jax.device_put(tgt_batched, NamedSharding(mesh, P(STAGE_AXIS, *([None] * len(tgt_rest)))))
216        head_loss = model.head_loss_fn(p.head, h_dist, tgt_dist)
217        return head_loss + aux_total / M
218 
219    return jax.value_and_grad(full_loss)(params)

226def _zero_bubble_body(
227    params: PipelineParams,
228    tokens: jax.Array,
229    targets: jax.Array,
230    *,
231    model: PipelineModel,
232    num_stages: int,
233    num_microbatches: int,
234    hidden_shape: tuple[int, ...],
235    hidden_dtype: jnp.dtype,
236) -> tuple[jax.Array, PipelineParams]:
237    """Explicit forward + split-backward pipeline, run inside a shard_map.
238 
239    The backward is split into two passes that the schedule decouples:
240 
241    - **B pass** (reverse time): per timestep compute ``dx`` (cotangent w.r.t.
242      the stage input) and ppermute it upstream. This is the critical path that
243      unblocks stage ``s-1``; it also seeds the embed gradient on stage 0.
244    - **W pass**: replay the stored output cotangents through ``vjp_p`` to get the
245      weight gradient ``dp``. W depends on nothing downstream, so it is fully
246      deferred here (and on hardware is slotted into the warmup/cooldown bubbles).
247 
248    B and W use *separate* vjp closures so the deferred W is genuine compute, not
249    a buffered add — that decoupling is what makes the bubble fillable.
250    """
251    sid = jax.lax.axis_index(STAGE_AXIS)
252    S = num_stages
253    M = num_microbatches
254    T = M + S - 1
255    fwd_perm = [(i, i + 1) for i in range(S - 1)]
256    bwd_perm = [(i + 1, i) for i in range(S - 1)]
257    is_first = sid == 0
258    is_last = sid == (S - 1)
259 
260    stage_params = jax.tree_util.tree_map(lambda x: x[0], params.stage)
261 
262    # Forward sweep: keep per-timestep vjp closures + outputs (static-keyed by t).
263    vjp_x_by_t: list[Any] = [None] * T
264    vjp_p_by_t: list[Any] = [None] * T
265    embed_vjp_by_t: list[Any] = [None] * T
266    out_by_t: list[Any] = [None] * T
267    tgt_by_t: list[Any] = [None] * T
268    valid_by_t: list[Any] = [None] * T
269 
270    buf = jnp.zeros(hidden_shape, hidden_dtype)
271    total_loss = jnp.zeros((), jnp.float32)
272    for t in range(T):
273        m = t - sid
274        valid = (m >= 0) & (m < M)
275        m_clip = jnp.clip(m, 0, M - 1)
276        tok_m = jax.lax.dynamic_index_in_dim(tokens, m_clip, axis=0, keepdims=False)
277        tgt_m = jax.lax.dynamic_index_in_dim(targets, m_clip, axis=0, keepdims=False)
278 
279        # Loop vars are bound as lambda defaults: jax.vjp traces each immediately,
280        # so the closures capture this iteration's tensors (B023-safe).
281        embedded, embed_vjp = jax.vjp(lambda e, _t=tok_m: model.embed_fn(e, _t), params.embed)
282        stage_in = jnp.where(is_first, embedded, buf)
283        # stage_fn returns (hidden, aux); vjp closures take a (d_hidden, d_aux) cotangent.
284        (stage_out, stage_aux), vjp_x = jax.vjp(lambda x, _sp=stage_params: model.stage_fn(_sp, x), stage_in)

4"""Per-device op order for the zero-bubble pipeline backward.
5 
6The zero-bubble pipeline runs a single SPMD program on every stage (see
7``pipeline.py``): a device with stage id ``sid`` processes microbatch
8``m = t - sid`` at timestep ``t``, for ``t`` in ``range(T)`` with
9``T = M + S - 1`` (``M`` microbatches, ``S`` stages). Every device issues the
10same ``T`` forward (F), ``T`` input-gradient (B) and ``T`` weight-gradient (W)
11ops; the warmup/cooldown diagonal is handled by masking, not by a per-stage op
12count, so the schedule is one ordered list reused on every device.
13 
14Dependencies that the order must respect:
15 
16- **F** is a forward sweep: ``F(t)`` ships its activation to the next stage,
17  consumed by ``F(t)`` there at the following wavefront step, so a device runs
18  ``F(0), F(1), ..., F(T-1)`` in time order.
19- **B** is the backward critical path: ``B(t)`` produces the input cotangent
20  ``dx`` and ppermutes it upstream, where the neighbour consumes it as its
21  ``B`` input. A device therefore runs ``B(T-1), B(T-2), ..., B(0)`` in reverse
22  time, and each ``B`` blocks on the cross-stage ppermute from the stage below.
23- **W** depends only on ``dy_by_t[t]`` (produced by ``B(t)``) and a forward vjp
24  closure. It has no cross-stage and no cross-time dependency, so ``W(t)`` may
25  be issued at any point after ``B(t)``, and any interleaving of W into the B
26  stream is numerically identical (the grad accumulation is associative).
27 
28Schedule (ZB-H1 / Qi et al. "Zero Bubble Pipeline Parallelism", steady-state
291B1W): emit the forward sweep, then interleave one W after each B, offset by one
30slot so ``W(t)`` rides in the device-idle window of the *next* ``B`` while that
31``B`` waits on its upstream ppermute:
32 
33    F(0) ... F(T-1)                          # forward sweep, time order
34    B(T-1)                                    # warmup: no W ready yet
35    B(T-2) W(T-1)  B(T-3) W(T-2) ...          # steady 1B1W: W fills the B gap
36    B(0)   W(1)
37    W(0)                                      # cooldown drain
38

40slots without a partner, so on the per-device timeline the B-critical-path
41ppermute stalls are covered by W everywhere except the very first B.

44slots (the ``S - 1`` invalid diagonal slots do no real work but, being masked,
