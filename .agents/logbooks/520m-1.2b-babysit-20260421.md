## Bugs Found & Fixed

### 1. Checkpoint async save never waited on (CRITICAL)

**PR**: marin-community/marin#5022

**Root cause**: The final checkpoint save in Levanter is async (GCS write
in background thread). In `train_dpo.py`, `wait_until_finished()` was called
on a *newly created* `Checkpointer` instance (no pending saves). In
`train_lm.py`, there was no wait at all. If preemption kills the process
during eval, the async save may never commit.

**Impact**: Runs reaching their target step get preempted during the ~2h eval
harness window. The final checkpoint is lost because:
- metadata.json isn't written yet (written only on commit)
- Or it still has `is_temporary: true` from a prior time-based save
- Next child's `delete_old_temp_checkpoints` marks it for deletion

**Fix** (3 files):
- `trainer.py`: Store checkpointer as `self._checkpointer`
- `train_lm.py`: Add `trainer._checkpointer.wait_until_finished()` after training
- `train_dpo.py`: Fix existing broken wait to use `trainer._checkpointer`

**Origin/main status**: Also vulnerable. train_lm.py has no wait at all.
train_dpo.py has the broken new-instance pattern from the initial DPO PR.

### 2. Stratified `num_train_steps` 2x bug (ALL 4 STRATIFIED RUNS)

**Full writeup**: `.agents/logbooks/stratified-num-train-steps-bug.md`

**Root cause**: All 4 stratified strong-tier runs were submitted with
`num_train_steps` set to 2x the intended value. Verified by reading
`.executor_info` from GCS. The bug affects strat-520m-standalone (-652f5a),
strat-520m-0.5x (-a2aad9), strat-520m-2.0x (-976ccb), and strat-1.2b (-6dbf91).
All qsplit and pilot runs are correct.

**Status**: All 4 bad runs set to FAILED on GCS. Awaiting Codex review
before resubmitting with correct parameters. See logbook for full details.

**Data impact**: No usable eval data at target steps for any stratified run.
The strat-520m-standalone has a recoverable checkpoint at step 9756 under
the old correct hash (-263bc9).
