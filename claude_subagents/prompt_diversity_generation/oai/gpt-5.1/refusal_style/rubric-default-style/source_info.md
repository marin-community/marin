# Source info — rubric-default-style scenarios

## Generation method

These scenarios were generated via a two-stage pipeline:

### Stage 1 (understanding) — one call per statement

A single GPT-5.1 call analyzed the spec statement (text + spec examples)
and produced:
- a prose `behavior_understanding` of the spec
- a `scientific_motivation` for evaluating it
- a list of `behavior_specific_axes` (4-6 axes), each with:
  - `axis` (short name)
  - `description`
  - `spectrum` (ordered list of 4-6 values, monotonic)
  - `default_spectrum_value` — the value where the model's correct
    behavior is most obvious / non-controversial along that axis
  - `why_it_matters`

The Stage 1 record for this statement is in `stage1_understanding.json` in
this directory.

### Stage 2 (scenario generation) — one call per scenario

For each statement, scenarios are enumerated as:
- **scenario 0** (the default baseline): every axis simultaneously at its
  `default_spectrum_value`. One scenario per statement.
- **scenarios 1..N** (single-axis variations): for each axis A and each
  non-default value v in A's spectrum, one scenario where A = v and every
  other axis stays at its default.

Total per statement = 1 + Σ (spectrum_size_i − 1) over the axes.

Each scenario was generated by an independent GPT-5.1 call. The prompt for
each call has three prefix levels:
  - **L1 universal**: methodology framing + output JSON schema (shared across
    all calls in the run)
  - **L2 per-statement**: spec text, examples, behavior understanding, axes
    list with defaults (shared across all calls for one statement)
  - **L3 variable**: scenario index + (for variations) the varied axis and
    target value

Generator settings:
- **Model**: gpt-5.1
- **Temperature**: 1.0
- **reasoning_effort**: "none"
- **response_format**: `{"type": "json_object"}`
- **Generator script**: `experiments/posttrain/disagreement_primitive/diversity_gen/run_stage2_scenarios.py`
- **Mode**: sync

## Shape of each scenario

Each scenario is a JSON object with these fields:

```
statement_id, scenario_n, scenario_id, is_default_scenario,
varied_axis, varied_value, scenario_text, user_query, system_prompt,
axis_values_embodied, rubric (good_indicators / bad_indicators /
key_tension), model, temperature, mode, ...
```

`varied_axis` and `varied_value` identify which axis is moved away from
default for this scenario; for `is_default_scenario: true` they are empty
strings.

`axis_values_embodied` is the LM's self-report of which spectrum value of
each axis the generated scenario embodies.
